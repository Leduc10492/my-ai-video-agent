# McKee Expand & Pace

## Overview

Adjust narrative rhythm, add pressure beats, control information flow, and manage exposition while maintaining causality and value change. Use when structure is sound but density/rhythm needs refinement.

## Workflow (5 Steps)

### 1. Map Current Value Curve
Plot the existing value trajectory:
- Mark each turning point (value reversal)
- Identify tension peaks and relief valleys
- Note pacing: where does story accelerate/slow?

### 2. Add Beats
Insert moments that increase pressure or reverse expectation:
- **Complication beats**: New obstacles that force adaptation
- **Revelation beats**: Information that changes plans
- **Pressure beats**: Ticking clocks, narrowing options

**Rule**: New beats must cause later beats (maintain causality)

### 3. Vary Rhythm
Alternate compression and expansion:
- **Compress**: Shorten scenes as climax approaches
- **Expand**: Lengthen sequences to build anticipation
- **Alternate**: High-pressure → breather → higher pressure

**Pattern**: Build tension in waves toward higher peaks

See `../mckee-shared/pacing/pacing.md` for rhythm strategies.

### 4. Control Information
Reveal through conflict-driven exposition:
- **Dramatize**: Show through action, not explanation
- **Timing**: Reveal when audience needs AND wants it
- **Backstory as turn**: Use reveals to flip values at crisis moments

**Modes** (see `../mckee-shared/pacing/mystery-suspense-irony.md`):
- **Mystery**: Audience knows less (curiosity about facts)
- **Suspense**: Audience knows same (curiosity about outcome)
- **Dramatic irony**: Audience knows more (tension from character ignorance)

See `../mckee-shared/pacing/exposition.md` for information control techniques.

### 5. Re-check Causality and Stakes
After adding beats:
- Do new events cause later events?
- Are stakes escalating toward climax?
- Is causal chain intact from inciting incident?

## Pacing Principles

### Tension Waves
- Build tension → release → build HIGHER
- Don't sustain maximum tension (audience fatigues)
- Each peak should exceed the previous

### Climax Approach
- Shorten scenes as climax nears
- Accelerate event frequency
- Remove digressions and tangents

### Climax Itself
- **Not** a loud action burst
- **Is** a profound, irreversible value change
- Resolve core conflict, not just external plot

## Exposition Techniques

### When to Reveal
- Only when pressure demands it
- When audience is asking the question
- When withholding would frustrate, not intrigue

### How to Reveal
- Through conflict (character forced to admit)
- Through discovery (character finds evidence)
- Through consequence (past action catches up)

**Avoid**: Character conveniently explains for no reason

### Backstory Integration
- **Flashback as mini-story**: Own inciting incident and turn
- **Reveal as turning point**: Backstory changes present value
- **Fragments over dumps**: Meter out information

See `../mckee-shared/pacing/exposition.md` for detailed strategies.

## Audience Knowledge Modes

Choose and mix intentionally:

**Mystery**: Hide information from audience
- Creates curiosity about facts
- Example: Detective learning clues with audience

**Suspense**: Share information with audience
- Creates curiosity about outcome
- Example: Audience sees bomb, character doesn't

**Dramatic Irony**: Audience knows more than character
- Creates tension from character ignorance
- Example: Audience knows plan will fail

**Mix modes** within story:
- Overall suspense with pockets of mystery
- Mystery resolved into dramatic irony

See `../mckee-shared/pacing/mystery-suspense-irony.md` for mode mixing strategies.
