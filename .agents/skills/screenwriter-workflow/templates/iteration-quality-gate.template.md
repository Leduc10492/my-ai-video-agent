## 通过/失败检查

| 检查项 | 状态 | 证据 | 必要动作 |
| --- | --- | --- | --- |
| P0 Story Bone ready |  | 10 个骨骼字段足以支持结构稿 | 失败则阻止完整剧本 |
| P0 故事脊柱完整 |  | 激励事件、危机、高潮、结局都存在 | 失败则阻止导出 |
| P1 主引擎有效 |  | 场景推进依赖已声明的主引擎，不靠随机新设定 | 失败则导出前必须修订 |
| P1 结构漂移受控 |  | 外部行动没有抢走原本的情绪机制 | 失败则导出前必须修订 |
| P1 因果链完整 |  | 主要场景都由前一层压力导致 | 失败则导出前必须修订 |
| P1 主角能动性 |  | 主角采取不可逆行动，而不只是等待 | 失败则导出前必须修订 |
| P1 另一方能动性 |  | 远方/缺席角色也做出有代价的选择 | 失败则导出前必须修订 |
| P1 版权距离 |  | 无受保护名称、对白、精确场景顺序或独特术语 | 失败则必须修远 |
| P2 时长密度 |  | 估算时长在目标内，或偏差有理由 | 失败则删减或扩展 |
| P2 motif payoff |  | 重复物件/台词在高潮或结局里物理兑现 | 薄弱则修订 |
| P2 Screenwriter 风格 |  | 可见/可听动作，无小说式内心描写 | 薄弱则润色 |
| P3 对白/生活质感 |  | 主要角色有可区分的句式、回避方式、沉默和普通反应 | 薄弱则润色 |
| P3 生产可行性 |  | 场景、道具、VFX负担符合阶段假设 | 薄弱则记录风险 |
| P3 对比材料处理 |  | 对比材料标明状态，仅用于抽象功能/节奏/密度检查 | 不清楚则修订报告 |

状态值:

- `pass`
- `pass_with_note`
- `revise`
- `blocked`

## 精简 Anti-AI 检查

| 检查 | 状态 | 证据 | 修订方向 |
| --- | --- | --- | --- |
| 角色替作者讲主题 |  |  | 改成行动、回避、误解、失败反应。 |
| 对白过聪明 |  |  | 降低概括能力，保留笨拙、打断、误会和半句话。 |
| callback 过工整 |  |  | 让重复物件或句子服务动作，不做展示式押题。 |
| 人物太会表达自己 |  |  | 用沉默、转移话题、身体动作和错误决定表达。 |
| 冲突解决太干净 |  |  | 保留代价、余波或未解的生活压力。 |
| 外部行动压过情绪机制 |  |  | 减少新任务或设定，让外部行动回到关系压力。 |

## 迭代决策

从以下固定决策中选择一个:

- `revise_structure`: 创建下一版 `01_script_v{N+1}.md`，因为 Story Bone、主引擎、结构漂移、故事脊柱、因果、能动性或结局存在 P0/P1 问题。
- `expand_density`: 创建下一版 `01_script_v{N+1}.md`，因为结构方向正确但场景密度过低。
- `cut_runtime`: 创建下一版 `01_script_v{N+1}.md`，因为有效扩密后草稿超过目标时长。
- `polish_dialogue`: 创建下一版 `01_script_v{N+1}.md` 或定向场景修订，因为结构、密度和时长可接受，但生活质感、声音、沉默、未说出口的话或动作行工艺薄弱。
- `export_docx`: 仅在所有 P0/P1 检查通过且 P2 缺口可接受后导出。
- `hold_current`: 不创建新剧本版本，因为剩余问题不值得再生成一个 artifact 版本。
