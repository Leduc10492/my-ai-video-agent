# Changelog Format

Append newest entries near the top of `deliverables/0_admin/changelog.md`, below the format note.

## Template

```markdown
## [YYYY-MM-DD] - <Stage>

### Added
- `<file>`: what was created

### Changed
- `<file>`: what changed

### Archived
- `<old file>` -> `<archive path>`
```

## Rules

- Mention affected artifact IDs when useful.
- Mention upstream changes that require regeneration.
- Keep entries short and operational.
- Do not paste full generated content into the changelog.
